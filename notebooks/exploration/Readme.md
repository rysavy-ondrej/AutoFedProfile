# TLS Traffic Exploratory Analysis  
**Benign (WinApps) and Malware TLS Datasets**

This repository contains two exploratory data analysis (EDA) notebooks:

- `winapps.ipynb` – Benign TLS traffic generated by Windows applications  
- `malware.ipynb` – TLS traffic generated by malware samples grouped by family  

The goal is to **characterize structural, temporal, and protocol-level properties of TLS communication** before applying profiling or anomaly detection methods.

---

# Overview of Performed Analysis

## 1. Dataset Structure & Coverage
- Total number of flows
- Time span and temporal coverage
- Schema validation and missing value inspection

Purpose: Ensure dataset consistency and understand scope.

---

## 2. Distribution of Entities

### Benign dataset
- Connections per application
- Application activity over time

### Malware dataset
- Connections per malware family
- Family imbalance
- Temporal activity per family

Purpose: Identify dominant entities, imbalance, and campaign-like behavior.

---

## 3. Categorical Feature Diversity

Computed cardinality of TLS attributes such as:
- TLS version  
- Cipher suites  
- Extensions  
- SNI  
- JA3 (if available)

Also evaluated diversity per application / per family.

Purpose:
- Measure behavioral stability vs variability
- Detect polymorphism or infrastructure diversity

---

## 4. TLS Protocol Characteristics

- TLS version distribution
- Cipher suite usage
- Extension presence patterns

Purpose: Characterize cryptographic posture and protocol behavior.

---

## 5. Flow-Level Statistics (Robust Metrics)

For each dataset, computed:

- Duration (median, IQR, p95, p99)
- Bytes up / down
- Packets up / down
- Byte ratio (up/down)
- Inter-arrival time statistics

Because TLS traffic is heavy-tailed, robust metrics are preferred over mean/std.

---

## 6. Structural Indicators

- % short connections (<1s)
- % large transfers (>1MB)

Purpose: Summarize traffic composition and tail contribution.

---

## 7. Outlier Exploration

- Percentile-based inspection of extreme flows
- Tail behavior characterization

Purpose: Understand anomaly-prone regions of feature space.

---

# Suggested Extended Analysis

To deepen the exploration:

### Comparative Analysis
- Direct benign vs malware distribution comparison
- Feature separability assessment

### Entropy Metrics
- Entropy of SNI, cipher suites, extensions
- Stability vs diversity quantification

### Temporal Drift
- Rolling window statistics
- Distribution divergence across days

### Tail Contribution
- Volume concentration analysis (top 1% flows)
- Heavy-tail quantification

### GREASE Impact
- Cardinality before/after GREASE removal
- Effect on fingerprint stability

### Clustering
- GMM / k-means to discover communication modes
